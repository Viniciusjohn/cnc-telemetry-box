Certifique-se de que o Telemetry está rodando neste PC:
- Use scripts\start_telemetry.bat ou serviço equivalente.
- Verifique http://localhost:8001/healthz neste PC (tem que retornar HTTP 200).

IP detectado automaticamente: 192.168.56.1
Porta do Telemetry: 8001

De outro computador na mesma rede (ou do host, se a VM estiver em modo bridge), teste:
PowerShell:
    Invoke-WebRequest -Uri "http://192.168.56.1:8001/healthz"

Se LOCAL funciona e de OUTRO HOST não funciona:
- Verifique se o firewall do Windows está liberando a porta 8001/TCP para a rede privada.
  (Exemplo: New-NetFirewallRule -DisplayName "CNC Telemetry 8001" -Direction Inbound -Protocol TCP -LocalPort 8001 -Action Allow -Profile Private)
- Confirme que o host remoto está na mesma faixa de IP.
- Anote se o outro host recebeu HTTP 200 ou o erro exibido.
